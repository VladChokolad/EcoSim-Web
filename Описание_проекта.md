# ВВЕДЕНИЕ

В современном мире компьютерное моделирование экосистем играет важную роль в изучении биологических процессов, образовании и научных исследованиях. Данная курсовая работа посвящена разработке веб-приложения для симуляции динамической экосистемы с использованием объектно-ориентированного программирования и библиотеки Three.js для 3D-визуализации.

Цель работы – создание интерактивной симуляции, моделирующей взаимодействие различных видов существ (травоядных и хищников) в замкнутой среде с учетом таких факторов, как питание, размножение, энергетический баланс и ограничения территории.

Основные задачи проекта:

1. Разработка архитектуры приложения с четким разделением ответственности между компонентами.
2. Реализация физической модели движения существ с учетом столкновений и границ мира.
3. Создание системы искусственного интеллекта для принятия решений существами (поиск пищи, избегание опасностей, размножение).
4. Визуализация 3D-сцены с использованием Three.js, включая текстурирование, освещение и тени.
5. Разработка интуитивного пользовательского интерфейса для управления симуляцией и отображения статистики.
6. Обеспечение производительности и оптимизации для плавной работы в реальном времени.

Проект реализован на языке JavaScript с применением принципов объектно-ориентированного программирования, что позволяет легко расширять функциональность и добавлять новые типы существ.

# ГЛАВА 1. АНАЛИЗ ТЕХНОЛОГИЙ И ПОСТАНОВКА ЗАДАЧИ

## 1.1. Обзор существующих решений

Существует множество решений для моделирования экосистем, от простых 2D-симуляций до сложных 3D-моделей. Популярные платформы включают NetLogo, GAMA и специализированные научные пакеты. Однако большинство из них требуют установки дополнительного программного обеспечения и не предоставляют веб-интерфейса.

Веб-приложения для симуляции экосистем встречаются реже, но обладают преимуществами доступности и кроссплатформенности. Примеры включают проекты на базе WebGL, такие как "Evolution Simulator" и "Primordial Life". Анализ показал, что многие из них имеют ограниченную функциональность или закрытый исходный код.

## 1.2. Выбор технологического стека

Для реализации проекта выбран следующий технологический стек:

- **HTML5/CSS3** – для структуры и стилизации веб-интерфейса.
- **JavaScript (ES6+)** – основной язык программирования, обеспечивающий логику приложения.
- **Three.js** – библиотека для работы с WebGL, предоставляющая удобный API для создания 3D-графики.
- **Canvas API** – для генерации текстур и дополнительной графики.
- **Встроенные API браузера** – для обработки событий, анимации и управления состоянием.

Выбор Three.js обусловлен его популярностью, хорошей документацией и поддержкой современных браузеров. Библиотека позволяет создавать сложные 3D-сцены с минимальными усилиями.

## 1.3. Постановка задачи

Требуется разработать веб-приложение, которое:

1. Создает 3D-сцену с текстурированной поверхностью, представляющей различные биомы (пустыня, сухая трава, трава, лес).
2. Населяет сцену динамическими объектами:
   - Растения (источник пищи для травоядных).
   - Травоядные (олени, зайцы) – питаются растениями, размножаются, прячутся в пещерах.
   - Хищники (волки, медведи) – охотятся на травоядных, размножаются.
3. Реализует поведенческие модели:
   - Движение с учетом физики и столкновений.
   - Поиск пищи/целей на основе расстояния.
   - Механизмы размножения при достижении порога энергии.
   - Система энергии – уменьшается со временем, пополняется при питании.
4. Предоставляет пользовательский интерфейс для:
   - Управления симуляцией (пауза, сброс, добавление существ).
   - Настройки скорости симуляции.
   - Отображения статистики популяции в реальном времени.
   - Просмотра детальной информации о выбранных существах.
   - Визуализации графиков изменения популяции.
5. Обеспечивает интерактивность:
   - Возможность выбора существ для просмотра информации.
   - Режим размещения новых существ щелчком мыши.
   - Управление камерой (вращение, масштабирование, перемещение).

# ГЛАВА 2. АРХИТЕКТУРА ПРИЛОЖЕНИЯ

## 2.1. Структура проекта

Проект состоит из следующих файлов:

- `index.html` – основной HTML-документ, содержащий структуру страницы, стили и подключение скриптов.
- `simulation.js` – основной JavaScript-файл, содержащий всю логику симуляции, классы и функции.
- `css/styles.css` – файл стилей (отсутствует в текущей версии, стили встроены в HTML).

Проект не требует сборки или компиляции, работает непосредственно в браузере.

## 2.2. Компоненты системы

Архитектура приложения построена на основе объектно-ориентированного подхода. Основные классы:

1. **Cave** – представляет пещеру, в которой могут прятаться травоядные. Ограничивает количество существ внутри.
2. **Plant** – модель растения, содержит энергию, визуализируется как цилиндр.
3. **Creature** – абстрактный класс для всех существ. Определяет общие свойства: энергия, скорость, положение, состояние.
4. **Herbivore** – класс травоядных, наследуется от Creature. Реализует поиск растений и поведение убегания от хищников.
5. **Predator** – класс хищников, наследуется от Creature. Реализует поиск травоядных и атаку.
6. **Deer, Hare, Wolf, Bear** – конкретные подклассы, уточняющие параметры (скорость, энергия, внешний вид).

Дополнительные системные компоненты:

- **ThreeJS Manager** – отвечает за инициализацию сцены, камеры, рендерера, освещения и текстур.
- **UI Manager** – управляет элементами интерфейса, обновлением статистики, обработкой событий.
- **Simulation Loop** – главный цикл анимации, обновляющий состояние всех объектов.
- **Chart Manager** – строит графики популяции с использованием Canvas API.

## 2.3. Взаимодействие компонентов

Взаимодействие компонентов можно представить следующей схемой:

```
Пользовательский интерфейс → Обработчики событий → Обновление состояния → ThreeJS рендеринг
```

1. Пользователь взаимодействует с кнопками или canvas.
2. Обработчики событий изменяют состояние симуляции (добавление существ, пауза и т.д.).
3. Главный цикл анимации обновляет позиции и состояния всех объектов.
4. ThreeJS перерисовывает сцену с учетом новых данных.
5. UI Manager обновляет статистику на экране.

Все объекты хранятся в глобальных массивах `plants`, `herbivores`, `predators`, `caves`. Это упрощает управление, но в будущем может быть рефакторинг в менеджер сущностей.

# ГЛАВА 3. РЕАЛИЗАЦИЯ ОСНОВНЫХ ФУНКЦИЙ

## 3.1. Графический движок и визуализация

Инициализация Three.js происходит в функции `initThreeJS()`. Создается сцена с фоном цвета `0x0a0a1a` и туманом для глубины. Камера – перспективная, с начальной позицией (30,25,30). Рендерер настраивается с включенными тенями и антиалиасингом.

Освещение состоит из:
- AmbientLight (рассеянный свет) интенсивностью 0.6.
- DirectionalLight (направленный свет) интенсивностью 0.8, отбрасывающий тени.

Поверхность земли – плоскость 120x120 единиц, текстурированная сгенерированной картой биомов. Текстура создается функцией `generateBiomeTexture()`, которая использует Canvas API для рисования градиентов, имитирующих пустыню, сухую траву, траву и лес.

Дополнительно добавлен GridHelper для визуальной ориентации.

## 3.2. Моделирование поведения существ

Каждое существо наследует от класса `Creature`, который содержит базовую логику:

- **Движение**: Использует вектор скорости, корректируемый в зависимости от наличия цели. Если цель отсутствует, существо блуждает случайным образом. Учитываются границы мира (`worldBounds`).
- **Энергетическая система**: Энергия уменьшается со временем (0.3 единицы в секунду). При достижении нуля существо умирает. При питании энергия увеличивается.
- **Размножение**: При достижении порога энергии (80 для травоядных, 150 для хищников) и случайной проверке существо может размножиться. Энергия родителя уменьшается, создается потомок со случайными вариациями скорости.
- **Поиск цели**: Травоядные ищут ближайшее растение, хищники – ближайшего травоядного. Алгоритм перебирает массив целей и выбирает ближайшего в радиусе видимости.
- **Питание**: При столкновении с целью (расстояние < 1.2) существо "съедает" её, получая энергию. Растение или травоядное удаляется из симуляции.

Пещеры (`Cave`) предоставляют укрытие для травоядных. В каждой пещере может находиться до 3 травоядных одновременно. При приближении хищника травоядные могут забежать в пещеру, если есть свободное место.

## 3.3. Пользовательский интерфейс

Интерфейс разделен на две основные области:

1. **Область 3D-визуализации** (`canvas-container`) – занимает большую часть экрана, отображает сцену Three.js.
2. **Панель управления** (`ui-panel`) – расположена справа, содержит:
   - Статистику популяции по видам (олени, зайцы, волки, медведи).
   - Кнопки управления: Пауза, Сброс, Добавление существ.
   - Слайдер скорости симуляции (0.1x – 3x).
   - Кнопку открытия графика популяции в полном размере.

Все элементы стилизованы с использованием CSS с градиентами, тенями и анимациями. Интерфейс адаптирован для мобильных устройств (медиа-запросы).

Дополнительные элементы:
- **Модальное окно графика** – отображает линейный график изменения популяции за последние 100 кадров.
- **Панель информации о существе** – появляется при клике на существо, показывает детальные параметры (тип, энергия, скорость, состояние). Панель можно перетаскивать.

## 3.4. Управление симуляцией

Управление реализовано через обработчики событий:

- **Клавиатура**: Клавиши WASD и стрелки перемещают камеру, Space – пауза, R – сброс.
- **Мышь**: 
  - Левый клик по существу – открытие панели информации.
  - Левый клик по земле в режиме размещения – создание нового существа.
  - Правый клик + перемещение – вращение камеры (через OrbitControls).
  - Колесико – масштабирование.
- **Интерфейсные кнопки**: Привязаны к функциям управления симуляцией.

Главный цикл анимации использует `requestAnimationFrame` для плавного обновления. Время между кадрами (`deltaTime`) учитывается для независимости от частоты кадров.

# ГЛАВА 4. ТЕСТИРОВАНИЕ И РЕЗУЛЬТАТЫ

## 4.1. Функциональное тестирование

Тестирование проводилось вручную в браузерах Chrome, Firefox и Edge. Проверялись следующие сценарии:

1. Запуск симуляции – все объекты создаются, анимация начинается.
2. Взаимодействие травоядных и растений – травоядные находят и съедают растения, энергия восстанавливается.
3. Охота хищников – хищники преследуют и съедают травоядных.
4. Размножение – при достаточной энергии появляются новые существа.
5. Пещеры – травоядные могут прятаться, хищники не могут зайти.
6. Пользовательский интерфейс – кнопки, слайдеры, график работают корректно.
7. Производительность – при большом количестве объектов (>100) FPS остается приемлемым.

Выявленные проблемы:
- Иногда существа застревают на границах.
- При высокой скорости симуляции коллизии обрабатываются менее точно.
- Панель информации о существе может перекрывать важные элементы.

Проблемы были частично устранены корректировкой параметров движения и добавлением ограничений.

## 4.2. Производительность и оптимизация

Для обеспечения плавной работы применены следующие оптимизации:

- **Отслеживание FPS** – с помощью Stats.js отображается текущая частота кадров.
- **Ограничение количества объектов** – при превышении лимита (например, >200) старые объекты удаляются.
- **Оптимизация рендеринга** – используются стандартные материалы, геометрия с умеренным количеством полигонов.
- **Эффективное обновление** – массивы объектов фильтруются при удалении, чтобы избежать накопления "мертвых" ссылок.

На среднестатистическом компьютере симуляция стабильно работает при 60 FPS с 50-80 объектами одновременно.

## 4.3. Результаты работы приложения

В результате разработки создано полностью функциональное веб-приложение, удовлетворяющее поставленным задачам. Приложение позволяет:

- Наблюдать за динамикой экосистемы в реальном времени.
- Вмешиваться в процесс, добавляя или удаляя существ.
- Анализировать статистику популяции через графики.
- Изучать детали отдельных существ.

Проект демонстрирует применение принципов ООП в JavaScript, использование Three.js для 3D-графики и создание интерактивного пользовательского интерфейса.

# ПРИЛОЖЕНИЕ 1. ИСХОДНЫЙ КОД ПРОЕКТА

Исходный код проекта состоит из двух основных файлов:

## index.html

Содержит разметку страницы, встроенные стили и подключение внешних библиотек (Three.js, OrbitControls, Stats.js). Определяет структуру интерфейса: заголовок, контейнер для 3D-сцены, панель управления, футер, модальные окна.

## simulation.js

Основной скрипт размером ~1500 строк. Включает:

- Константы и глобальные переменные.
- Классы: Cave, Plant, Creature, Herbivore, Predator, Deer, Hare, Wolf, Bear.
- Функции инициализации Three.js, генерации текстур, создания пещер.
- Главный цикл анимации `animate()`.
